# Frontend Design System

A Claude Code skill that eliminates AI slop from frontend design. Opinionated typography, color, layout, animation, and conversion rules informed by 57 peer-reviewed papers on prompt engineering.

## What This Does

When you ask Claude Code to build any web interface, this skill activates and enforces:

- **Banned font list** — Inter, Roboto, Poppins, and 8 other AI-default fonts are blocked
- **Banned color palettes** — Purple gradients, purple+teal, generic blue SaaS schemes are blocked
- **Banned layout patterns** — Cookie-cutter card grids, 3-column icon+title+paragraph sections
- **Required standards** — CSS custom properties, semantic HTML, `prefers-reduced-motion`, responsive down to 320px
- **Self-verification checklist** — 20+ AI slop detection checks + 20+ quality standard checks run before delivery

The result: every interface looks like it was designed by a senior designer, not generated by an AI.

## Install

```bash
claude install-skill https://github.com/edmondyv/frontend-design-system
```

Or manually clone and point Claude Code at it:

```bash
git clone https://github.com/edmondyv/frontend-design-system.git
claude --plugin-dir ./frontend-design-system
```

## Trigger Phrases

The skill activates when you say things like:
- "Build a landing page"
- "Create a website"
- "Design a frontend"
- "Make a web app"
- "Build a dashboard"
- "Make it look good"
- "Remove AI slop"
- "Fix the styling"

## Philosophy

AI models naturally converge toward safe, predictable designs — the "AI slop" aesthetic. This skill forces divergence at every decision point: font choice, color palette, layout structure, animation approach.

The standard: Awwwards winners and the best Framer templates are the floor, not the ceiling.
